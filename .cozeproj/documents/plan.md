# OfflineChat 离线聊天应用 - 重构计划

## 概述

OfflineChat 是一款基于局域网的离线聊天移动应用，适用于飞机、动车等无网络场景。一台设备开热点并启动服务端，其他设备连接热点后自动发现服务端并加入聊天。支持端到端加密、内容伪装、可选密码保护、申请制审批等功能。

**平台**：Mobile (React Native / Expo 54)

## 技术方案

| 维度 | 选择 | 理由 |
|------|------|------|
| 通信协议 | WebSocket (ws) | 实时双向通信，局域网低延迟 |
| 服务端发现 | UDP 广播 | 自动发现服务端，无需手动输入 IP |
| 加密 | 混合方案 | 公共消息用共享密钥，私聊/小群用 ECDH |
| 本地存储 | AsyncStorage | 消息持久化，退出后可查看历史 |
| 导航 | Stack | 线性流程，无 Tab Bar |
| 样式 | Tailwind CSS (Uniwind) | 项目模板已有 |

## 架构简化

**核心变更**：一个服务器 = 一个大房间

| 之前 | 之后 |
|------|------|
| 多个房间，需创建/加入 | 一个服务器 = 一个聊天室 |
| 6位房间号 | 可选8位密码 |
| 手动输入房间号 | 自动发现服务端 |
| 需要知道服务端 IP | 广播自动发现 |

## 功能模块

### 1. 服务端（Express + WebSocket）

**职责**：运行在主机设备上，提供聊天服务和自动发现

| 功能 | 说明 |
|------|------|
| WebSocket 服务 | 实时消息转发（群聊/私聊） |
| UDP 广播 | 定期广播服务端信息，供客户端发现 |
| 密码验证 | 可选8位密码，加入时验证，房主可运行时修改 |
| 申请制 | 可选，加入需填写理由，房主审批。可设置已批准用户过期时间（0=永不过期） |
| 在线用户管理 | 维护在线用户列表 |

### 5.3 小群管理

**群主产生**：谁创建小群，谁就是群主

**群主权限**：

|------|------|

- 群主被踢出/审批过期 → 自动转让群主给第一个成员
- 小群继续存在，不影响其他成员

**僵尸群清理**：
- 小群超过 X 天没有新消息（默认7天，可配置）→ 自动解散
- 或群主手动解散

### 5.4 私聊防骚扰

- 拉黑列表存储在本地，可以取消拉黑

**与房主黑名单的区别**：

### 7. 加密与伪装

**混合加密方案**：

| 消息类型 | 加密方式 | 谁能解密 |
|---------|---------|---------|
| **公共消息** | 共享密钥（所有人同一个密钥） | 所有人（包括房主） |
| **私聊** | ECDH sharedKey（每对用户独立密钥） | 只有双方 |
| **小群消息** | groupKey（小群共享密钥） | 只有小群成员 |

**加密流程**：
```
发送：明文 → 加密(根据消息类型选择密钥) → 伪装编码 → WebSocket
接收：WebSocket → 伪装解码 → 解密 → 明文显示
```

**密钥管理**：
- 公共消息密钥：房主启动服务器时生成，用户加入时获得
- 私聊密钥：ECDH 密钥交换，每对用户独立计算
- 小群密钥：群主生成，通过 ECDH 安全分发给成员

**伪装模式**：
| 模式 | 伪装效果 |
|------|----------|
| 天气预报 | `☀️ 晴 22°C 湿度45% [加密数据]` |
| 代码日志 | `// log: [加密数据]` |
| 购物清单 | `- 苹果 x1 [加密数据]` |
| 系统日志 | `[SYS] heartbeat ok [加密数据]` |
| 无伪装 | 直接显示加密文本 |

### 6. 本地持久化

**存储内容**：
- 昵称
- 已连接的服务端列表（服务端ID + 地址 + 名称）
- 每个服务端的消息历史（按服务端ID隔离）
- 加密/伪装设置

**存储结构**：
```typescript
// AsyncStorage 存储 key
@chat:nickname              // 用户昵称
@chat:servers               // 已连接的服务端列表
@chat:messages:{serverId}   // 每个服务端的消息历史（隔离存储）
@chat:settings              // 加密/伪装设置

// 数据结构
interface StoredServer {
  id: string;               // 服务端唯一ID (UUID)
  address: string;          // IP:端口
  name?: string;            // 服务端名称
  lastConnected: number;    // 上次连接时间
}
```

**Server ID 策略**：
- 服务端首次启动时生成 UUID 作为 serverId
- serverId 持久化到本地文件 `./server-id.txt`
- 服务端重启时读取相同的 serverId，保证客户端可以匹配历史记录
- 服务端删除后重新创建时生成新的 serverId

**行为**：
- 消息实时写入本地，按服务端ID隔离
- 切换服务端时，加载对应服务端的历史消息
- 不同服务端的消息完全隔离，不会混淆
- 退出 App 后再进入可看历史
- 新消息追加到对应服务端的历史中

**历史记录管理**：
- 客户端保存已连接过的服务端列表

### 7. 设备身份与安全

**用户身份识别**：
- 客户端生成 `deviceId`（UUID），存储在 `expo-secure-store`（绑定设备，无法导出）
- 同一设备在所有服务器上使用同一个 deviceId
- 服务端通过 deviceId 识别已批准用户

**挑战-响应验证**：
- 客户端首次启动时生成 `deviceSecret`（随机密钥），存入 `expo-secure-store`
- 连接时：客户端发送 deviceId → 服务端返回 challenge → 客户端用 deviceSecret 签名 → 服务端验证
- 防止 deviceId 被盗用：盗用者没有 deviceSecret，无法通过验证

**依赖**：`expo-secure-store`（Expo 官方库，支持三端）
- 聊天室顶部可切换查看其他服务端的历史消息（只读模式）
- 只读模式下可以阅读和删除历史消息，但不能发送新消息
- 顶部显示"只读模式"提示

**服务端重启处理**：
- 服务端重启后 serverId 不变（从本地文件读取）
- 客户端断线后自动重连，获取相同的 serverId
- 客户端加载本地历史消息，不受服务端重启影响
- 服务端内存中的消息丢失，但客户端本地历史保留

### 7. 断线重连

**重连机制**：
| 功能 | 说明 |
|------|------|
| 自动重连 | WebSocket 断开后自动尝试重连（指数退避：1s → 2s → 4s → 8s，最大30s） |
| 状态恢复 | 重连成功后自动重新加入聊天室，无需手动操作 |
| 消息恢复 | 从本地存储加载历史消息，断线期间的消息在重连后同步 |
| 服务器记忆 | 记住上次连接的服务器地址，下次启动自动尝试连接 |
| 离线提示 | 断线时顶部显示"连接已断开，正在重连..."红色提示条 |
| 重连成功提示 | 重连成功后显示"已重新连接"绿色提示条，3秒后消失 |

**重连流程**：
```
断线 → 显示"正在重连..." → 尝试重连(指数退避) → 成功 → 自动加入 → 同步消息 → 显示"已重新连接"
                                                    ↓ 失败(超过最大次数)
                                              显示"连接失败" → 返回首页
```

## 实施步骤

### 阶段一：原型设计

1. **原型设计** — 加载 `design-canvas` 技能，按 mobile 平台规范设计以下页面原型：昵称设置页、服务端发现与连接页、聊天室页（含房主控制面板、在线用户侧边栏、小群功能）。原型完成后提示用户确认，调用 done 工具提交等候用户确认。

2. **重构服务端** — 简化为单房间架构，添加可选密码验证（支持运行时修改）、申请制审批（含过期策略）、踢人功能、黑名单管理、UDP 广播发现、设备身份验证、小群管理。关键文件：`server/src/chatManager.ts`、`server/src/index.ts`、新增 `server/src/broadcast.ts`

4. **重构前端页面** — 删除创建/加入房间页，新建服务端发现连接页，重构聊天室页（集成房主控制面板、申请审批、踢人功能、黑名单管理、小群功能、本地持久化）。关键文件：`client/screens/chatRoom/index.tsx`、新增 `client/screens/connect/index.tsx`

6. **实现通知系统** — 集成 expo-notifications，使用系统默认提示音，实现可配置的通知功能（提示音、震动、本地通知、审批通知、应用内横幅）。关键文件：新增 `client/utils/notifications.ts`
